Identidad bespoke por materia para los toolkits de StudyVaults

Módulo A del rediseño: hoy las 6 materias comparten un único launcher genérico en /[vault]/herramientas — mismo grid de cards, cero identidad. La propuesta: cada materia arma su propio launcher con motivo visual e iconografía propios, pero todas las herramientas siguen abriendo dentro del mismo runner compartido. Este documento muestra 3 launchers representativos.

FIG. 02 — Patrón

El launcher es bespoke. El runner es uno solo.

Al hacer click en cualquier card, el usuario sale del launcher temático y entra a un shell neutro compartido por las 6 materias: mismo breadcrumb de vuelta, mismo jump-nav horizontal para saltar entre herramientas, mismo layout de contenido. Probá — hacé click en cualquier card de arriba y mirá cómo se actualiza este sketch.

El launcher cambia de materia a materia (motivo, íconos, color de acento); el runner que envuelve cada herramienta individual no cambia nunca — así el usuario reconoce la materia por el launcher, pero nunca se pierde dentro de una herramienta.
